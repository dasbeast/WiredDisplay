# WiredDisplay 1.0.4

WiredDisplay 1.0.4 is a major polish release focused on cursor responsiveness, receiver behavior, windowing, and day-to-day usability. This version continues the move toward a more stable "plug it in and use it" experience while keeping the more experimental cursor overlay path available for testing.

## Highlights

- Smoother wired display sessions with major work on sender-side cursor tracking and transport behavior.
- Improved receiver window behavior so the app acts more like a normal macOS app window.
- Better sender diagnostics, including a dedicated "Stats for Nerds" window.
- Cleaner logging and more explicit pipeline refresh visibility during long-running sessions.

## New And Improved

### Receiver-side improvements

- Improved receiver rendering and frame presentation behavior for more reliable long-running playback.
- Cleaned up receiver session and listener flow to make startup and reconnect behavior more predictable.
- Receiver window management now behaves more like a standard app window instead of a special-case utility surface.
- Added a proper `DisplayReceiver` app icon so the receiver shows up correctly in the Dock and app switcher.

### Sender-side improvements

- Added a dedicated receiver-side cursor overlay mode for lower-latency cursor motion.
- Added dynamic cursor shape mirroring support for more natural pointer behavior when the side overlay path is enabled.
- Improved cursor handoff, click-and-hold behavior, pointer ownership, and general cursor tracking stability across the branch.
- Added sender-side cursor pipeline refresh behavior for long sessions, including visible refresh logging.
- Reduced noisy capture logging so recurring issues like frame gaps no longer flood the console.
- Simplified the sender transport story: video continues to use the stable TCP path, while the low-latency UDP side channel is reserved for cursor motion when the overlay path is enabled.

### UI and workflow improvements

- Moved "Stats for Nerds" out of the main sender UI and into a separate window opened from the `View` menu.
- Added a small `BETA` tag to the cursor overlay toggle so experimental functionality is clearly labeled.
- Updated sender controls and diagnostics to better reflect the current transport and cursor paths.
- Improved release and workflow plumbing for shipping builds.

## Feature Notes

### Receiver-side cursor overlay

WiredDisplay 1.0.4 includes a receiver-side cursor overlay mode designed to reduce perceived cursor latency by sending cursor motion outside of the normal video path. When enabled, the sender streams the display over TCP as usual, while cursor motion can use a dedicated high-frequency side channel.

This is the most important new feature in 1.0.4, but it is still considered experimental. It is intended for users who want the lowest possible cursor latency and are comfortable testing an in-progress path.

### Stats for Nerds window

Diagnostics are now available in a separate window instead of living on the main sender page. This keeps the primary UI cleaner for normal use while preserving live session stats for testing, debugging, and tuning.

## Known Issues

- `Use Side Cursor Overlay` is still in beta. Cursor polling and cursor refresh behavior have improved significantly, but the overlay path can still degrade or feel laggy during very long sessions on some setups.
- In rare cases, the cursor overlay path may still need a pipeline refresh to recover from lag even when the main video stream remains healthy.
- The separate stats window is intended for diagnostics and should be treated as a support tool rather than part of the core streaming flow.
- Video transport is TCP-only in the current production path. The remaining UDP path is reserved for cursor motion and related overlay work, not general video streaming.
- Because WiredDisplay depends on private virtual display APIs, behavior can still vary across macOS updates and hardware combinations.

## Recommended Positioning For This Release

WiredDisplay 1.0.4 is a stability-and-polish release. The core display streaming path is more mature, the receiver behaves more like a proper Mac app, and diagnostics are easier to access. The low-latency cursor overlay is now available for users who want it, but it should still be presented as a beta feature in this version.
